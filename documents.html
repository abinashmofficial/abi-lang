Introduction

AbiLang is a progressive, lightweight, and versatile scripting language named after you (Abinash). Built from scratch using TypeScript, it runs natively on both backend environments via CLI terminal engines and in web browsers through an interactive sandboxed playground.

Core Philosophy: AbiLang is designed to minimize boilerplates, eliminate unnecessary tokens like declarations (const, let) and mandatory semicolons, and introduce robust structural clarity with clean braces and optional parameter parentheses.

Key Features

  • No Declarations: Assign variables directly. The interpreter handles allocations dynamically.
  • Flexible Flow Control: Clean loop iterations and conditional checks with optional parentheses.
  • Integrated Framework Mechanics: Native controller configurations, model definitions, route bindings, and view render engine helpers.
  • Cloud Ready: Native cloud integrations featuring asynchronous fetch routines and json_parse parsing out of the box.

Installation & CLI

Run the AbiLang compiler CLI and local web playground directly on your machine. Choose the appropriate installer command for your environment below.

macOS & Linux Setup

To run the single-click remote automated installation script, copy the command below:

# Fetch and execute bash auto-installer script
curl -fsSL https://raw.githubusercontent.com/abinashmofficial/abi-lang/main/install.sh | bash

Manual Compilation & Local Run

If you prefer a manual setup, clone the repository, install its packages, and boot up the visual playground:

# 1. Clone repository
git clone https://github.com/abinashmofficial/abi-lang.git
cd abi-lang

# 2. Install TypeScript development tooling
npm install

# 3. Build compiler engine binaries
npm run build

# 4. Spin up local interactive web playground
npm run web

Once running, navigate to http://127.0.0.1:8080 to access the sandbox interface locally.

Variables

Variables in AbiLang store data dynamically. You do not need keywords like var, let, or const, and semicolons are completely optional.

name = "Abinash"
age = 21
pi = 3.14159
print "Name: " + name
const name = "Abinash";
let age = 21;
const pi = 3.14159;
console.log("Name: " + name);
name = "Abinash"
age = 21
pi = 3.14159
print("Name: " + name)
$name = "Abinash";
$age = 21;
$pi = 3.14159;
echo "Name: " . $name;
Tip: Dynamic typing ensures variables can change their underlying representation on successive operations automatically.

Conditionals

Conditionals allow branching execution. AbiLang utilizes modern block scoping via braces {} while making parentheses around expressions optional.

score = 85
if score >= 90 {
    print "Grade A"
} else if score >= 80 {
    print "Grade B"
} else {
    print "Grade F"
}
let score = 85;
if (score >= 90) {
    console.log("Grade A");
} else if (score >= 80) {
    console.log("Grade B");
} else {
    console.log("Grade F");
}
score = 85
if score >= 90:
    print("Grade A")
elif score >= 80:
    print("Grade B")
else:
    print("Grade F")
$score = 85;
if ($score >= 90) {
    echo "Grade A";
} elseif ($score >= 80) {
    echo "Grade B";
} else {
    echo "Grade F";
}

Loops

Iterate over code ranges efficiently. AbiLang supports standard while loops and native array iterations without parenthesis boilerplates.

count = 1
while count <= 3 {
    print "Count: " + count
    count = count + 1
}
let count = 1;
while (count <= 3) {
    console.log("Count: " + count);
    count = count + 1;
}
count = 1
while count <= 3:
    print("Count: " + str(count))
    count = count + 1
$count = 1;
while ($count <= 3) {
    echo "Count: " . $count;
    $count = $count + 1;
}

Functions

Functions isolate modular calculations. AbiLang uses the func keyword to declare reusable code blocks that accept arguments and return evaluations.

func greet(name) {
    return "Hello, " + name + "!"
}
message = greet("Abinash")
print message
function greet(name) {
    return "Hello, " + name + "!";
}
const message = greet("Abinash");
console.log(message);
def greet(name):
    return "Hello, " + name + "!"
message = greet("Abinash")
print(message)
function greet($name) {
    return "Hello, " . $name . "!";
}
$message = greet("Abinash");
echo $message;

Controllers & Models (OOP Concepts)

AbiLang utilizes **Object-Oriented Programming (OOP)** paradigms by leveraging the dynamic nature of dictionary literals and lexical closures. Since AbiLang focuses on a clean syntax without boilerplate class decorators, objects are instantiated using **Factory Constructor Patterns**.

OOP Architecture in AbiLang:
  • Objects: Created as dynamic dictionaries ({}) containing both attributes and member functions.
  • Encapsulation: Private fields can be initialized in constructor scopes; inner functions capture these states in their closures.
  • Inheritance: Accomplished by initializing parent object dictionaries and merging/extending them with new attributes or overridden functions.

Defining OOP Models

In AbiLang, models represent database entities and domain objects. You can structure them as constructor classes using factory functions:

# file: model/user_model.abi

# Factory constructor for UserModel
func UserModel(id, name, email) {
    # 1. Instantiate the object dictionary
    self = {}
    self.id = id
    self.name = name
    self.email = email

    # 2. Define member methods (encapsulation)
    func get_info() {
        return "User: " + self.name + " (" + self.email + ")"
    }
    self.get_info = get_info

    func update_email(new_email) {
        self.email = new_email
        return self
    }
    self.update_email = update_email

    return self
}

To use this model in your application:

include("model/user_model.abi")

# Create user instances
user1 = UserModel(1, "Abinash", "abinash@example.com")
user2 = UserModel(2, "Developer", "dev@example.com")

print user1.get_info()  # Outputs: User: Abinash (abinash@example.com)
user1.update_email("abinash.official@example.com")
print user1.get_info()  # Outputs: User: Abinash (abinash.official@example.com)

Controller Classes & OOP Inheritance

Controllers manage application logic, coordinating between models and views. You can implement controller classes that inherit base behavior through dictionary compositions:

# file: controller/base_controller.abi

# Base Controller Class/Factory
func BaseController(controller_name) {
    self = {}
    self.name = controller_name

    # Shared rendering utility method
    func render_view(view_name, data) {
        print "[BaseController] Preparing layout context for: " + self.name
        return view(view_name, data)
    }
    self.render_view = render_view

    return self
}

Next, inherit from the BaseController to construct specific controller implementations:

# file: controller/user_controller.abi
include("controller/base_controller.abi")
include("model/user_model.abi")

# Derived UserController Class/Factory
func UserController(user_id) {
    # 1. Inherit attributes and methods from BaseController
    self = BaseController("UserController")
    self.user_id = user_id

    # 2. Define controller actions (derived methods)
    func show() {
        # Fetch data using UserModel
        user = UserModel(self.user_id, "Abinash", "abinash@example.com")
        
        # Invoke inherited render_view method
        return self.render_view("profile", user)
    }
    self.show = show

    return self
}

Finally, register the controller action in your routing system:

# file: routes/route.abi
include("controller/user_controller.abi")

# Routing to OOP Controller Instance Action
func handle_request(req_id) {
    controller = UserController(req_id)
    return controller.show()
}

Routing & Views

AbiLang features standard routing structures built into the language engine, allowing you to define route schemas that map directly to controller actions.

Route Registration

Register routes using the native route statement, specifying HTTP verb, endpoint, target controller action, and route alias:

# file: routes/route.abi
include("controller/controller.abi")

route("get", "/profile", "controller@show_profile", "user.profile")

Returning Views

The built-in view("filename") helper automatically loads and processes the corresponding user interface definition from the project layout space:

# file: controller/controller.abi
func index() {
    # Automatically resolves to view/index.ui template
    return view("index")
}

AbiLang vs JavaScript

While AbiLang is built on top of JavaScript/TypeScript runtimes, there are substantial differences in their syntactic execution structures.

JS Variable Declarations

JavaScript strictly requires lexical identifiers such as let, const, or var to allocate variables. Forgetting these in strict modes triggers runtime errors. In AbiLang, declarations are completely optional: variables are dynamically created upon assignment.

JS Module Imports

JavaScript uses ES modules (import/export) or CommonJS (require) to manage dependencies, which frequently result in scoping issues or bundler errors. AbiLang uses a simplified include("path") system that executes sub-scripts recursively in the parent global scope, offering an easy shared state model.

AbiLang vs Python

Python is a script-oriented programming language, but AbiLang differs in its approach to structuring code blocks and target platforms.

Py Block Scoping vs Whitespace

Python relies on strict indentation (spaces/tabs) to define code blocks, which can lead to layout errors during formatting or copy-paste actions. AbiLang resolves block contexts using brackets ({}), creating robust, formatting-independent structures that look clean and readable.

Py Web Browser Runtime Integration

Python requires massive web-assembly runtime interpreters (e.g., Pyodide) to run inside web browsers, which slows down initial page loads. AbiLang compiles to lightweight JavaScript bundles natively, executing in the browser with minimal overhead.

AbiLang vs PHP

PHP has historically powered backend routing, but AbiLang optimizes script formatting and built-in architecture setups.

PHP Clean Syntax (No Sigils)

PHP requires variables to begin with a dollar sigil (e.g., $variable), adding syntactic noise to arithmetic and logical statements. AbiLang removes this noise, keeping variable references clean and readable (e.g., variable).

PHP Built-In Framework Model

PHP requires a web server (Nginx/Apache) or external frameworks (Laravel) to register routes and views. AbiLang integrates these concepts natively into the language kernel, permitting routing tables and controller resolutions straight out of the box.
