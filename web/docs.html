About AbiLang

AbiLang is a progressive scripting language designed for clean structure, optimal execution speed, and simplicity. It allows writing expressive scripts with optional parentheses, clean brackets, and zero boilerplate code.

Why is it unique?
  • Hybrid Architecture: Compiles and runs directly in Node.js on the backend, or bundles to clean standard JavaScript for runtime performance in web browsers.
  • Procedural Scoping: Imports other scripts recursively using the global include("path") system, keeping functions, models, and constants within a shared environment.
  • Cloud Fetch: Built-in async HTTP call methods (fetch) and JSON parsing utilities (json_parse) for smooth REST-database connectivity.
Minimum System Requirements
  • macOS: macOS 10.15+ (Catalina), Node.js v18.0.0+, npm v9.0.0+
  • Linux: Ubuntu 20.04 LTS+ / Debian 11+ / Fedora 36+, Node.js v18.0.0+, Git installed
  • Windows: Windows 10/11 (64-bit), PowerShell 5.1+ / PowerShell Core 7+, Node.js v18.0.0+, npm v9.0.0+
  • Mobile: Android SDK API Level 30+ (Android 11+), Xcode 14+ (macOS only, for iOS), CocoaPods installed

1. Default Handler (handlers/handler.abi) 
include("entities/entity.abi")

class Handler {
    public func index() {
        e = Entity()
        print "Calling entity: " + e.data()
        return screen("index")
    }

    public func docs() {
        return screen("docs")
    }
}
2. Creating and Extending Handlers

Handlers use object-oriented class syntax and support public, private, and protected visibility modifiers for methods. Member data can be assigned to this. To define dynamic route handlers, map paths to class action methods:

include("entities/entity.abi")

class ProfileHandler {
    public func show_profile() {
        this.log_profile_view()
        return screen("profile")
    }

    private func log_profile_view() {
        print "Accessing user profile..."
    }
}
1. Route Registrations (navigation/routes.abi) 
include("handlers/handler.abi")

route("get", "/", "handler@index", "home")
2. Returning Screens with the screen() Method

Use the built-in screen("filename") helper inside your handler actions. The screen parser resolves filenames to screens/filename.ui automatically, so you don't have to specify folders or file extensions.
